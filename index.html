Introduction to NLP

What is NLP? Language forms, Turing test, phases, applications

What is NLP?

Natural Language Processing (NLP) is a field of computer science that focuses on making artificial agents understand and work with human languages using computational techniques to read, interpret, and generate language in a way that feels natural to people.

NLP is interdisciplinary — it combines Computational Linguistics, AI, Statistics, Mathematics, and Signal Processing.

Computational Linguistics (CL)

Scientific study of language using computational methods. Tries to understand how language is structured and how humans use it.

NLP vs CL

NLP focuses on creating products using CL that help artificial agents understand/interpret/generate language. In practice, CL and NLP terms are used interchangeably.

Classification of Languages

Type Subtypes Examples
Natural Languages Based on language family Hindi, Bengali, Tamil, English, Manipuri
Constructed Languages Artificially designed Esperanto (1887, L.L. Zamenhof), Klingon (Star Trek), Dothraki, Na'vi
Programming Languages Low/Mid/High-level Assembly, C, Python, JavaScript
Not all human languages are natural! Esperanto (1887) was created for easy cross-cultural communication. Klingon was created for Star Trek (1967). These are likely exam questions!

Three Forms of Language

📝 Text

  • Written form using characters/symbols
  • Set of symbols = Script
  • Only 40% of world languages have a script
  • One language → multiple scripts (e.g., Manipuri uses Bengali-Assamese and Meitei Mayek)
  • One script → multiple languages (e.g., Latin script: English, French, Spanish)
  • Encoded via ASCII, ISCII, UTF-8

🎙 Speech

  • Time-varying acoustic waveform (pressure variations)
  • Features: Pitch, Loudness, Speed
  • Sampled at fixed rate (e.g., 16 kHz)
  • Frames of 20–30 ms assumed quasi-stationary
  • Represented using MFCC (Mel-Frequency Cepstral Coefficients)
  • Prosodic features: pitch, energy, duration

🤚 Gesture

  • Body movement (spatio-temporal stream)
  • Sequence of poses over time
  • Types: Hand, body posture, facial, head movement
  • Sensors: RGB/Kinect cameras, accelerometers, data gloves
  • Pipeline: Input → Preprocessing → Keypoint Detection → Feature Extraction → Temporal Modeling → Classification

The Turing Test & CAPTCHA

Turing Test (Alan Turing, 1950)

Tests whether a machine can exhibit intelligent behaviour indistinguishable from a human.

  • 3 terminals: 1 computer + 2 humans, physically separated
  • Text-only communication
  • Human interrogator asks both the human respondent and AI respondent questions
  • If interrogator cannot distinguish AI from human → AI passes
  • ELIZA (1966) was the first chatbot said to pass the Turing test
Originally called the "Imitation Game"

Chinese Room Argument (John Searle, 1980)

Criticism of the Turing Test.

  • An English speaker in a room follows rules to respond in Chinese without understanding Chinese
  • Looks intelligent from outside, but has no understanding inside
  • Conclusion: Passing Turing Test ≠ genuine intelligence
  • Led to: Weak AI (simulates understanding) vs Strong AI (truly thinks like a human)

Reverse Turing Test → CAPTCHA

Here a machine determines if the responder is human or bot. Purpose: prevent spam, fake accounts, DoS attacks.

CAPTCHA = Completely Automated Public Turing test to tell Computers and Humans Apart. Tasks easy for humans, hard for bots (distorted text, image selection, pattern recognition).

Other Turing-like Tests

Test Who What it tests
Lovelace Test 2.0 (2014) Mark O. Riedl Creativity — AI produces artifacts exceeding designer's intentions
Coffee Test (2010) Steve Wozniak General intelligence — robot makes coffee in an unfamiliar house
Ebert Test (2011) Roger Ebert Can a synthesized voice tell jokes with human timing/inflection?

Ambiguity in Language

Why is processing language so complex? Types of ambiguity you MUST know.

Why is Language Processing Complex?

  • Patterns change with language, person, context, locality
  • New words appear constantly (selfie, ghosted, influencer)
  • Word meanings shift over time: mouse (animal → computer), cloud (sky → storage), gay (joyful → LGBTQ+), villain (farm worker → bad guy)
  • Ambiguity: word/phrase/sentence has more than one meaning due to flexible, context-dependent nature of language

1. Lexical Ambiguity

A single word has multiple meanings. Caused by:

Homonymy

Same form/sound, different unrelated meanings and origin

  • bank → financial bank OR river bank
  • bat → animal OR cricket bat

Polysemy

Same word, multiple related meanings

  • star → celestial body OR celebrity
  • cricket bat vs baseball bat (both related to sports)

2. Syntactic Ambiguity

Grammar allows more than one valid structure. The same words in the same order can be parsed differently.

Sentence Interpretation 1 Interpretation 2
"I saw the man with the telescope" I used a telescope to see The man had a telescope
"The old men and women are evacuated" Both old men and women Only the men are old
"The small flower garden" The garden is small The flowers are small
Prepositional phrases, adverbial phrases, and relative clauses are common sources of syntactic ambiguity because they can attach to either the noun or the verb.

3. Semantic Ambiguity

Structure is clear, but the meaning is not. Even grammatically correct sentences have unclear intended meaning.

Example Ambiguity
"Amal called Bimal because he was tired" Who is 'he'? Amal or Bimal? (pronoun reference)
"I went to the bank" Financial bank or riverbank? (homonymy)
"Only she told him that she loved him" Scope of 'only' changes meaning completely
"Flying planes can be dangerous" Act of flying (gerund) OR planes that are flying (participle)
Gerund vs Participle is a common exam question: Gerund (-ing) acts as a NOUN ("Swimming is fun"). Participle (-ing) acts as an ADJECTIVE ("Swimming in the river, he felt refreshed").

Combined Syntactic + Semantic Ambiguity

When both structural AND meaning ambiguity occur together, NLP becomes extremely complex. A single Telugu sentence like "vaadu koncem gelichina pettukunnaadu" can have 4+ different meanings because 'koncem' (small amount) can modify different parts AND 'pettu' can mean keep/bet/wear.

This is why NLP is hard — natural language is flexible, context-dependent, and inherently ambiguous at multiple levels simultaneously.

Text Normalization

Converting text into a standard, consistent format — with all the Python code

What is Text Normalization?

Process of converting text into a standard, consistent format to improve quality and make it easier for machines to process. Creates an equivalence class of units.

Why needed? Natural language is highly inconsistent: color vs colour, THE vs The vs the, can't vs cannot, thnx vs thanks, etc.

Without normalization, ML models misinterpret variations of the same word, leading to lower accuracy in search engines, chatbots, etc.

Technique 1: Case Folding

Convert mixed-case text to one case (usually lowercase).

text = "Hello World!"
normalized_text = text.lower()
# Output: "hello world!"

True-casing: Restoring correct capitalization (e.g., the → The at sentence start). Important for search engines, chatbots, translation.

Technique 2: Contraction Expansion

Expand shortened word forms to their full forms: can't → cannot, I'll → I will, I'm → I am.

import re
contractionDict = {"can't": "cannot", "I'm": "I am", "it's": "it is"}

def expandContraction(text, contractionDict):
    pattern = re.compile(r'\b(' + '|'.join(contractionDict.keys()) + r')\b')
    return pattern.sub(lambda x: contractionDict[x.group()], text)

print(expandContraction("I'm happy because it's a sunny day", contractionDict))
# Output: "I am happy because it is a sunny day"

Technique 3: Spelling Variations

Handle regional differences (colour → color, theatre → theater). Uses a predefined mapping dictionary.

spelling_variations = {"colour": "color", "theatre": "theater"}

def normalize_spelling(text, variations):
    words = text.split()
    return " ".join([variations.get(word, word) for word in words])

# Using TextBlob library
from textblob import TextBlob
corrected = str(TextBlob(text).correct())

Technique 4: Spelling Correction

Correct typos (definately → definitely, recieved → received).

from spellchecker import SpellChecker
spell = SpellChecker()
text = "I definately recieved the anouncement yesturday."
words = text.split()
corrected_text = " ".join([
    spell.correction(word) if word in spell.unknown(words) else word
    for word in words

# Fuzzy matching approach
from fuzzywuzzy import process
best_match = process.extractOne("accomodate", ["accommodate", "recommend"])
Algorithms used: Levenshtein edit distance, Damerau-Levenshtein, Jaccard similarity, Soundex. The Viterbi algorithm is a probabilistic extension of minimum edit distance.

Technique 6a: Stemming

Stemming reduces words to their root/stem by chopping off endings. The stem may NOT be a valid dictionary word. Fast but crude.

from nltk.stem import PorterStemmer, SnowballStemmer, LancasterStemmer

words = ["running", "flies", "better", "mice", "cakes"]

porter = PorterStemmer()
stems = [porter.stem(w) for w in words]
# ['run', 'fli', 'better', 'mice', 'cake']

snowball = SnowballStemmer("english")
stems = [snowball.stem(w) for w in words]

lancaster = LancasterStemmer()
stems = [lancaster.stem(w) for w in words]
# More aggressive: ['run', 'fli', 'bet', 'mic', 'cak']

# Regex Stemmer
from nltk.stem import RegexpStemmer
stemmer = RegexpStemmer('ing$|s$|e$|able$', min=4)

Porter Stemmer — The Algorithm (Exam Favourite!)

Developed by Martin F. Porter (1980). Words = Stem + Suffixes. Uses measure m = number of vowel-consonant (VC) sequences.

Word structure: [C](VC)ᵐ[V] — where m is the measure
Step What it does Example
1a Handles plural markers (sses, ies, s) caresses → caress, ponies → poni, cats → cat
1b Handles tense markers (-ed, -ing) running → run, filing → file
1c y → i if stem has vowel happy → happi (not sky → sky)
2 Replace regular suffixes (m>0) relational → relate, conditional → condition
3 Replace common suffixes (m>0) electrical → electric, hopeful → hope
4 Remove suffixes (m>1) revival → reviv, allowance → allow
5a,5b Clean up final -e or double consonants probate → probat, control → control
Limitations: Over-stemming (universe/university → univers), Under-stemming (keeping too much). Porter stemmer ignores context.

Technique 6b: Lemmatization

Lemmatization reduces words to their dictionary form (lemma) using vocabulary and morphological analysis. Slower but accurate.

from nltk.stem import WordNetLemmatizer
lemmatizer = WordNetLemmatizer()
words = ["running", "flies", "better", "mice"]
lemmas = [lemmatizer.lemmatize(word, pos="v") for word in words]
# ['run', 'fly', 'better', 'mouse']
Feature Stemming Lemmatization
Output Stem (may not be real word) Lemma (valid dictionary word)
Speed Faster Slower
Accuracy Lower Higher
Uses vocabulary? No Yes
Example: "better" better (unchanged) good (correct!)

Sentence Segmentation

Divides continuous text into individual sentences. Challenges: periods in abbreviations (Mr., p.m.), nested quotes, languages without explicit boundaries (Chinese, Japanese).

import re
text = "Hello!! How are you? Let's meet at 5 p.m. today in Mr. Idli"

# Basic (BAD — splits on abbreviations too)
re.split(r'[.!?]+', text)  # gives 7 wrong sentences

# Better — only split when punctuation is followed by space
re.split(r'[.!?]\s+', text)

# Keep the punctuation in output (lookbehind assertion)
re.split(r'(?<=[.!?])\s+', text)

# ML-based (best) using spaCy
import spacy
nlp = spacy.load("en_core_web_sm")
doc = nlp(text)
sentences = [sent.text for sent in doc.sents]
Lookbehind assertion (?<=pattern) checks what is on the LEFT without consuming characters. Lookahead (?=pattern) checks what is on the RIGHT.

N-gram Language Models

Frequency, MLE, Smoothing — the core of probabilistic NLP

What is a Language Model?

A model that assigns probabilities to sequences of characters/words. N-gram is the simplest such model.

An N-gram is a contiguous sequence of N items (characters or words).

N Name Example (word: "wishing")
1 Unigram wishing
2 Bigram wi, is, sh, hi, in, ng (character); "wishing you", "you a" (word)
3 Trigram wis, ish, shi, hin, ing (char); "wishing you a" (word)

Character N-gram Code

text = "Malayalam is a palindrome word"
text = text.lower()

# Character bigrams
ngrams = [text[i:i+2] for i in range(len(text)-1)]
print(ngrams)

# Count frequencies
from collections import Counter
ngramFreq = Counter(ngrams)
for ngram, freq in ngramFreq.items():
    print(f"'{ngram}': {freq}")

Word N-gram Code

text = "wishing you a good day"
words = text.split()

# Word bigrams
ngrams = [(words[i], words[i+1]) for i in range(len(words) - 1)]
# [('wishing', 'you'), ('you', 'a'), ('a', 'good'), ('good', 'day')]

from collections import Counter
ngramFreq = Counter(ngrams)
print(ngramFreq)

Relative Frequency

Relative Frequency = Count of unit / Total units in corpus
from collections import Counter
text = "The quick brown fox jumps over the lazy dog"
words = text.lower().split()
wordCounts = Counter(words)
totalWords = sum(wordCounts.values())

relativeFreq = {}
for word, count in wordCounts.items():
    relativeFreq[word] = count / totalWords
    print(f'{word} : {round(relativeFreq[word],3)}')
Limitations of frequency: Raw frequency doesn't allow fair comparisons across different corpus sizes. Zipf's Law only holds for large corpora.

Maximum Likelihood Estimation (MLE)

Estimates probability distribution of word sequences that maximizes likelihood of seeing the training data.

P(wₙ | wₙ₋₁) = Count(wₙ₋₁, wₙ) / Count(wₙ₋₁)

Example Calculation

Count("machine learning") = 50, Count("machine") = 150

Count("parrot learning") = 100, Count("parrot") = 110

P(learning | machine) = 50/150 ≈ 0.33
P(learning | parrot) = 100/110 ≈ 0.90
MLE's BIG PROBLEM: Assigns ZERO probability to unseen N-grams (words/phrases not in training data). Also overfits to training corpus and is sensitive to outliers.

Smoothing Techniques

Smoothing redistributes probability mass from seen events to unseen ones, so everything gets non-zero probability.

Add-1 (Laplace) Smoothing ▼

Add 1 to all N-gram counts. Simple baseline.

P_add-one(wₙ|wₙ₋₁) = (Count(wₙ₋₁, wₙ) + 1) / (Count(wₙ₋₁) + V)

V = vocabulary size. Problem: Distorts distribution when vocabulary is large — assigns too much probability to rare/unseen events.

Add-k Smoothing ▼
P_add-k(wₙ|wₙ₋₁) = (Count(wₙ₋₁, wₙ) + k) / (Count(wₙ₋₁) + k×V)

k is typically small (0.01, 0.1). Better than Add-1 as it allows controlled adjustment. Preferred in real scenarios.

Good-Turing Smoothing ▼

Key idea: Estimate probability of unseen events from behavior of rare events.

Adjusted count C* = (C + 1) × N_{C+1} / N_C

N_C = number of events occurring exactly C times (frequency-of-frequency / count-of-count).

P(unseen word) = N₁ / N

N₁ = words appearing exactly once, N = total corpus size.

Example

Words: ox:5, box:4, fox:2, cat:1, dog:1

N₁=2 (cat, dog), N₂=1 (fox), N₅=1 (ox)

C* for words with frequency 1 = (1+1)×N₂/N₁ = 2×1/2 = 1

If corpus N=1000, N₁=500: P(unseen) = 500/1000 = 0.5 (50% probability mass reserved for unseen!)

Good-Turing fails when N₁ is inaccurate/noisy, or when there isn't enough data to get reliable NC statistics.

Comparison Table

Method Key Idea Problem
MLE Relative frequency Zero probability for unseen, overfits
Add-1 (Laplace) Add 1 to all counts Distorts large-vocabulary distributions
Add-k Add small k to all counts Optimal k is corpus-dependent
Good-Turing Use rare events to estimate unseen Needs reliable count-of-count statistics

Morphological Analysis

Words, morphemes, word formation — the complete guide

What is Morphological Analysis?

The study of words, how they are formed, and their internal relationships. A morpheme is the smallest meaningful unit of a word (grammatical function or meaning).

  • It is a word or part of a word that has meaning
  • Cannot be divided further without creating meaningless remainders
  • Recurs with relatively stable meaning across different word environments
A morphological analyser takes a word as input and outputs: (1) minimum meaning-bearing units (morphemes) + (2) their grammatical relations.

Types of Morphology

Inflectional Morphology

Derives new words based on tense, mood, person, number, gender, case — WITHOUT changing the word's core meaning or class.

  • dogs = dog + -s (plural marker)
  • doing = do + -ing (tense marker)
  • marked = mark + -ed (past tense)
  • taller = tall + -er (comparative)

Inflectional morphemes: -s, -'s, -es, -ed, -en, -ing, -er, -est

Derivational Morphology

Creates NEW words from existing ones by adding derivational affixes. MAY change the word class and/or meaning.

  • readable = read + -able (verb → adjective)
  • exactly = exact + -ly (adj → adverb)
  • activity = active + -ity (adj → noun)
  • unhappy = un- + happy (negation)

Derivational morphemes: -ic, -ly, -ate, -ity, -ship, un-, re-

Root, Base, Stem & Lemma — Key Distinctions!

Term Definition Example
Root Most basic form carrying the core meaning. Cannot be broken further. May not stand alone. struct in construct/destruction
Base Form to which affixes can be added. May be a root OR a derived form. react (derived from act) is a base for reaction, reactive
Stem Output of stemming — crude heuristic removal of suffixes. Not always a real word. runn (from running, runner)
Lemma Output of lemmatization — proper dictionary/base form after morphological analysis. am/is/are → be; easily → easy
Root ≠ Base ≠ Stem ≠ Lemma — this is a common exam distinction! Know each one clearly.

Types of Morphemes

By Independence

  • Free morpheme: Can stand alone as a word. car, dog, do
  • Bound morpheme: Cannot stand alone, must attach. -s, un-, -ing

By Function

  • Inflectional: Grammatical variation without category change
  • Derivational: Creates new word, may change category

By Structure

  • Root: Core meaning-bearing unit
  • Affix: Attached to root (prefix, suffix, infix)

By Role

  • Lexical: Meaning-bearing (objects, actions). car, happy, run
  • Functional: Grammatical relations. -s, the, in, -ing

Word Formation Processes (All 10!)

1. Compounding, 2. Blending, 3. Clipping ▼

Compounding: Two+ words → new word. notebook (note+book), basketball, homemade, रेलगाड़ी

Blending: Parts of two words merged. Bollywood (Bombay+Hollywood), smog (smoke+fog), brunch (breakfast+lunch), motel (motor+hotel)

Clipping: Shortening without meaning change.

  • Back-clipping (front kept): ad (advertisement), exam (examination), gas (gasoline)
  • Fore-clipping (end kept): phone (telephone), varsity (university)
  • Middle-clipping: flu (influenza)
  • Complex-clipping: ma'm (madam), fax (facsimile)
4. Acronyms, 5. Back-formation, 6. Reduplication ▼

Acronyms: Initial letters pronounced as a word. JEE, NASA, CAPTCHA. Initialism: each letter pronounced (PMO, USA).

Back-formation: New word by removing perceived affix. edit (from editor), babysit (from babysitter).

Reduplication: Root/stem repeated. Three types in Indian languages:

  • Exact repetition: बार-बार (baːr-baːr), চলতে-চলতে
  • Rhyme (second is meaningless rhyme): চায়-বায় (tea-etc)
  • Partial prefix repetition: দা-দৰমহা
7. Borrowing, 8. Coinage, 9. Eponymy, 10. Affixation ▼

Borrowing: Adopting words from other languages. karaoke (Japanese), pizza (Italian), किताब/दफ्तर/साहब (Persian).

Coinage: Creating entirely new words. Google (brand → verb).

Eponymy: Brand/person name becomes common word. Sandwich (Earl of Sandwich), Teddy bear (President Roosevelt).

Affixation: Adding units to root/base.

  • Suffix: End. do+ing=doing, do+able=doable
  • Prefix: Beginning. un+happy=unhappy, re+write=rewrite
  • Infix: Middle (rare in Indo-Aryan). spoon+s+ful=spoonsful
  • Circumfix: Both beginning AND end simultaneously. en+light+en=enlighten
  • Duplifix: Partial/full root repetition (e.g., Pingelapese: mahuta → mahutamahuta)
  • Interfix: Connector between morphemes. speed+o+meter=speedometer

Morphological Ambiguity

A word analyzed in more than one valid way.

Type Example Explanation
Affix attachment unlockable un+lockable (not lockable) OR unlock+able (can be unlocked)
Category ambiguity -er suffix taller (comparative adj) OR doer (agentive noun)
False segmentation recover Looks like re+cover but means "get better" (single lexical unit)
Scope ambiguity unhappily un+happily OR unhappy+ly — where does negation apply?

Why Morpheme Analysis is Hard

Homonymous Morphemes

Same form, different function. -er in taller (comparative) vs teacher (agentive).

Allomorphs

Same grammatical function, different surface forms. Plural -s in: cats /s/, dogs /z/, judges /ɪz/.

Zero Morph (ϕ)

Grammatical meaning but no visible form. sheep+ϕ (plural), cut+ϕ (past tense).

Opaque Structure

Cannot split despite looking divisible. understand ≠ under+stand, department ≠ depart+ment, handsome ≠ hand+some.

Syllables

A syllable is the smallest unit of rhythm and pronunciation, built around one vowel sound.

Structure: Onset (initial consonants) + Nucleus (vowel) + Coda (final consonants)

Example: strength → onset:/str/ nucleus:/e/ coda:/ngth/

  • Phoneme: Smallest sound unit that distinguishes meaning (bat vs pat)
  • Diphthong: Two vowel sounds in one syllable (rain /reɪn/, out /aʊt/, coin /kɔɪn/)
  • Monosyllable: One syllable. Polysyllable: Multiple syllables.
  • Open syllable: Ends with vowel (to, hi, go). Closed syllable: Ends with consonant (art, fast)
Late, time, fine are diphthongs even with one vowel letter because the tongue GLIDES (sound changes). Seat, food are NOT diphthongs — the sound stays steady.

Regular Expressions

Formal notation, Python re module, JavaScript regex — complete guide with examples

What is a Regular Expression?

A formal, algebraic notation to define a search pattern for strings. Invented by Stephen C. Kleene (1951). Based on regular language theory.

Every RE has an equivalent Finite State Automaton (FSA). Thompson's construction: RE → ε-NFA → DFA (subset construction) → Minimized DFA.

Core Operators

Operator Symbol Meaning Precedence Example
Union | Choice between patterns Lowest ab|xy → ab or xy
Concatenation . Sequence of symbols Middle abc → abc
Kleene Star * Zero or more repetitions Highest ab* → a, ab, abb...

Extended/Derived Operators

Operator Symbol Meaning Rewritten as Example
Positive Closure + One or more aa* ab+ → ab, abb, abbb
Optional ? Zero or one a|ε ab? → a, ab
Exact count {n} Exactly n times — a{3} → aaa
Range {m,n} m to n times — a{2,4} → aa, aaa, aaaa

Metacharacters

Quantifiers

re.findall(r"c.t", "cat cut cot")  # . = any char
re.findall(r"ab+", "a ab abb")     # 1 or more
re.findall(r"ab*", "a ab abb")     # 0 or more
re.findall(r"colou?r", "color colour")# optional u
re.findall(r"\d{4}", "2000,2026")    # exactly 4 digits
re.findall(r"\d{4,}", "2000,20001") # 4 or more
re.findall(r"\d{2,4}", "20,200,2026")# 2 to 4

Anchors & Character Classes

re.search("^Hello", text)  # ^ = start
re.search("world$", text)  # $ = end
re.findall("[aeiou]", text) # char class
re.findall("[0-9]", text)   # digit range
re.findall("[^aeiou]", text) # NOT vowels
re.findall("cat|dog", text)  # pipe = OR
re.findall("(ab)+", text)   # grouping

Quantifier Types

Bounded vs Unbounded

  • Bounded: {n}, {m,n} — max repetition known in advance
  • Unbounded: *, +, {3,} — potentially infinite

Greedy vs Lazy vs Possessive

  • Greedy (default): +, *, ?, {n,m} — match as MUCH as possible, allow backtracking
  • Lazy: *?, +?, ??, {n,m}? — match as LITTLE as possible, expand if needed
  • Possessive: *+, ++, ?+, {n,m}+ — match as much as possible, NEVER backtrack
Possessive quantifiers prevent ReDoS (Regular Expression Denial of Service) attacks.

Python re Module — All Methods

import re
text = "Zebra has @bug in #bat. Found while playing with python and tom@cat."

re.match("cat", text)         # Match only at BEGINNING of string
re.search("cat", text).group() # Search ANYWHERE in string → first match
re.findall(r"\d+", text)       # Returns ALL matches as a list
re.sub(r"\d", "X", text)        # Replace all digits with X
re.split(r"[,;| ]", text)       # Split on comma, semicolon, pipe, space

Lookahead & Lookbehind

Zero-width assertions — check pattern without consuming characters.

Assertion Syntax Meaning
Positive Lookahead (?=pattern) Match if followed by pattern
Negative Lookahead (?!pattern) Match if NOT followed by pattern
Positive Lookbehind (?<=pattern) Match if preceded by pattern
Negative Lookbehind (?<!pattern) Match if NOT preceded by pattern 
# Mask first 7 digits of phone number
text = "Mobile number: 1234567890"
re.sub(r"\d(?=\d{3})", "*", text)

# Extract amount only if preceded by Rs.
text = "Paid Rs.1000 today and 200 yesterday"
re.findall(r"(?<=Rs\.)\d+", text)  # → ['1000']

Algebraic Properties of RE

JavaScript Regex

let text = "Zebra has @bug in #bat. Found while playing with python and tom@cat."

// test() — returns true/false
/bat/.test(text);

// search() — returns index position
text.search(/bat/);

// match() — returns matching strings
text.match(/\p{P}/gu);  // \p{P}=punctuation, g=global, u=unicode

// exec() — detailed match info, one at a time
let rgx = /bat/g; rgx.exec(text);

// replace()
text.replace(/\d(?=\d{3})/g, "*");  // mask phone digits

// split()
"apple,banana;orange|grape".split(/[,;|]/);

Property escapes in JS: \p{N}=Number, \p{P}=Punctuation, \p{Lu}=Uppercase, \P{N}=NOT number.

KEY DISTINCTIONS for exam: (ab) vs [ab] — parentheses group, brackets match ONE character from set. .* (greedy) vs .*? (lazy). (abc|ab)+ vs [abc]+ — first matches "abc" or "ab" repeated; second matches any char from {a,b,c} repeatedly.
